# Contributing

Thank you for your interest in contributing to the Personal Database Framework!

## How to Contribute

### Reporting Issues
- Use GitHub Issues to report bugs or suggest features
- Include clear description and steps to reproduce (for bugs)
- Check existing issues before creating new ones

### Submitting Changes
1. Fork the repository
2. Create a feature branch (`git checkout -b feature/your-feature`)
3. Make your changes
4. Test with Obsidian and/or your preferred AI tool
5. Commit with clear messages (`git commit -m "Add: description"`)
6. Push to your fork (`git push origin feature/your-feature`)
7. Open a Pull Request

### What We're Looking For
- **New entity schemas**: Templates for new types of data
- **Improved metadata**: Better AI instructions and inference rules
- **Documentation**: Clearer explanations, more examples
- **Integrations**: Guides for different AI tools (Claude, GPT, etc.)
- **Translations**: README and docs in other languages

### Style Guidelines
- Use consistent YAML formatting in frontmatter
- Keep metadata files focused and well-organised
- Include examples for new features
- Update README.md if adding major features

## Code of Conduct

Be respectful and constructive. We're all here to build useful tools.

## Questions?

Open an issue with the `question` label.
